How it works

You bring the agent. mykeep brings the capabilities.

mykeep has no AI of its own. Your agent — Claude Code, Cursor, anything that can call a local API — does the thinking; mykeep hands it capabilities it doesn't ship with, over a loopback connection on the stick. A stock agent becomes yours, on any machine, and stays private.

Private by design

Everything runs on the stick — no cloud round-trips, no telemetry, no accounts. Each capability and its data are encrypted at rest; lose the stick and a thief gets only ciphertext.

Works with any agent

Any client that can make a local HTTP call plugs in — Claude Code, Cursor, custom agents. A pasted snippet is the whole integration: no SDK, no plugin, no MCP required.

Carry it anywhere

The drive is the whole platform — binaries, config, and data all live on it. Plug into Windows, macOS, or Linux; pure-Go static binaries mean no installer, no runtime, no lock-in.

The suite

Four portable capabilities.

Each component is a self-contained capability — its own pure-Go binary and encrypted file — that any agent can use on its own, or all together behind one password. All four have shipped.

Capsule

portable encrypted memory

A single encrypted file you carry on the stick. Plug it into any machine and your agent's memory travels with you — semantic + keyword recall, never the cloud.

AES-256-GCM at rest local CPU embeddings vector + BM25 recall
GitHub MIT-licensed

Vault

a by-reference secrets broker

Your API keys live encrypted beside your memory. The agent makes authenticated calls by reference through a local broker, so the raw secret never reaches the model or the cloud.

key never seen by the agent SSRF-hardened egress approve + audit writes
GitHub MIT-licensed

Showstone

a portable browser it drives

A real Chromium your agent drives over a loopback REST API — reads pages as compact text, acts on numbered elements, and screenshots on demand. Sensitive actions block for your approval.

indexed-element snapshots sealed, portable profile approval gate + audit
GitHub MIT-licensed

Foundry

a sandboxed tool host

Runs tools — JavaScript in a WebAssembly sandbox — with only the capabilities you grant. It gives them an encrypted backend (kv, queue, cache, blobs) and a signed marketplace to install from.

WASM sandbox kv · queue · cache · blob capability-scoped
GitHub MIT-licensed
