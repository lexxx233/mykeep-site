How it works

You bring the agent. mykeep brings the capabilities.

mykeep has no AI of its own. Your agent does the thinking — Claude Code, Cursor, anything that calls a local API. mykeep hands it capabilities it doesn't ship with, so a stock agent becomes yours, on any machine, and stays private.

Private by design

Everything runs on the drive — no cloud, no telemetry, no accounts. Each capability and its data are encrypted at rest; lose the drive and a thief gets ciphertext.

Works with any agent

Any client that makes a local HTTP call plugs in — Claude Code, Cursor, custom agents. A pasted snippet is the whole integration: no SDK, no plugin, no MCP.

Carry it anywhere

The drive is the whole platform — binaries, config, and data live on it. Plug into Windows, macOS, or Linux; pure-Go static binaries mean no installer, no runtime.

Marketplace

A tool is a few lines of JavaScript.

No SDK, no build step. Write an async run(input) and call the foundry API for storage, HTTP, and authenticated requests. It runs in a WebAssembly sandbox with only the capabilities you grant — then ship it to the marketplace.

star-counter.js
// a complete mykeep tool async function run({ repo }) { // call GitHub as you, via Vault — the token never enters the tool const r = await foundry.vault.fetch({ credential: "github", url: `https://api.github.com/repos/${repo}`, }); const stars = JSON.parse(r.body).stargazers_count; await foundry.kv.set(repo, stars); // kept on the drive return { repo, stars }; }

Just JavaScript

The foundry global is the whole API — kv, queue, cache, blobs, brokered HTTP, Vault by reference, logs. No SDK, no plugin to install.

Sandboxed & scoped

Each tool runs in a WebAssembly sandbox under hard time and memory limits, reaching only the hosts and credentials a human grants it.

Reviewed & signed

Submit it and an AI reviewer screens the source; accepted tools are ed25519-signed and install on demand, verified against a pinned key.
